# Problem Memory Skill

记录编码过程中的问题与教训，通过持久化记忆和举一反三的推理，避免重复犯同样的错误。

## 核心理念

**每个错误只犯一次。** 当一个问题被识别和记录后，未来遇到相似场景时能够主动提醒和规避。

## 工作流程

### 1. 问题记录（Problem Logging）

当遇到以下情况时，**立即记录**：

- 编译错误、运行时错误、测试失败
- 逻辑漏洞、边界条件未处理
- 性能问题、内存泄漏
- 与预期不符的行为
- 任何让你花超过 2 分钟排查的"坑"

**记录格式：**

```yaml
- id: P001  # 自动递增编号
  date: 2026-05-15
  category: [编译|运行时|逻辑|性能|API|并发|类型|空安全|...]
  symptom: 问题的表面现象（错误信息/异常行为）
  root_cause: 根本原因（3-5句话）
  context: 发生时的条件（语言、框架、版本、特定场景）
  solution: 如何修复
  prevention: 未来如何避免
  related: [相关概念、类似问题]
  location: 代码位置/文件路径（如适用）
